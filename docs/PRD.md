# 產品需求文件 (PRD)：任務管理系統

## 1. 專案概述
- **背景與動機**：在日常生活中，學生、老師與一般大眾經常面臨繁雜的工作與課業任務。若缺乏有效的管理工具，容易導致任務遺漏、期限延宕，或無法清楚掌握各項任務的真實進度。此系統提供一個簡單、直覺的平台讓使用者存放、尋找，並且能夠有效區分任務的目前狀態。
- **目標用戶**：學生、老師、大眾。
- **核心價值主張**：提供直覺且清晰的任務狀態分類，能夠明確區分「代辦中」、「執行中」、「已完成」、「未完成」，讓使用者對所有任務進度一目了然，隨時掌握優先級與進度，有效提升個人生產力與時間管理能力。

## 2. 功能需求
系統將提供以下主要功能：

1. **使用者註冊與登入**
   - *使用者故事*：作為大眾，我希望能夠註冊並登入系統，以便安全地管理我個人的待辦清單並保留進度。
2. **新增與管理任務**
   - *使用者故事*：作為學生，我希望能建立新任務並設定初始狀態為「代辦中」或「執行中」，以便記錄我接下來必須完成的功課或報告。
3. **切換任務狀態**
   - *使用者故事*：作為老師，我希望能將正在進行的任務隨時更改狀態（如切換為「已完成」或「未完成」），以便準確反映任務當前的執行結果與學生的繳交狀況。
4. **任務狀態看板/全覽**
   - *使用者故事*：作為大眾，我希望能在主畫面上清楚看到任務依據（代辦中、執行中、已完成、未完成）分群顯示的列表，以便一眼掌握所有的工作進度和瓶頸。
5. **編輯與刪除任務**
   - *使用者故事*：作為學生，我希望能編輯任務內容或刪除不再需要/過期的任務，以便保持我的待辦清單整潔且符合最新狀況。

## 3. 非功能需求
- **技術限制**：
  - 後端框架：Python + Flask
  - 模板引擎：Jinja2
  - 資料庫：SQLite
- **效能與安全考量**：
  - 使用適當的安全機制來防範常見的安全漏洞（如：防護 SQL Injection, Jinja2 預設防護 XSS）。
  - 對受保護的路由實作驗證機制，保障個資安全。頁面載入時間應維持在 2 秒以內。

## 4. MVP 範圍

### Must Have (必備功能 - 初期實作)
- 使用者註冊、登入機制。
- 任務的新增、更新、刪除功能 ( CRUD )。
- 任務的四種狀態管理 (執行中、代辦中、已完成、未完成)。
- 個人首頁顯示任務列表。

### Should Have (應有功能 - 中期實作)
- 看板模式，將不同狀態的任務分列/分區塊顯示。
- 任務期限 (Due Date) 設置。

### Nice to Have (加分功能 - 未來擴充)
- 任務逾期自動轉為「未完成」狀態。
- 提醒通知、標籤功能與深色模式。
